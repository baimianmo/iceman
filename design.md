# Iceman Design & Architecture

Iceman 是一个基于 LLM 的私人银行客户关怀与自动化任务助手。它通过灵活的 Agent 架构与动态技能系统，实现从意图识别到复杂任务执行的全流程自动化。

## 1. 核心架构 (Core Architecture)

系统采用 **Router-Agent-Skill** 三层架构，通过统一的 `LLMClient` 接入层屏蔽底层模型差异（DeepSeek/GLM/Ollama），并支持动态扩展能力。

## 2. 核心组件 (Core Components)

### 2.1 接口层 (Interface Layer)
- **CLI (`main.py`)**: 提供命令行交互入口，支持单次执行与持续对话模式。
- **Feishu Server (`feishu_server.py`)**: 集成飞书/Lark 开放平台，处理消息回调、事件订阅与卡片消息发送。
  - 支持文本、图片、文件（PDF）等多种消息类型。
  - 实现消息去重与幂等处理。

### 2.2 代理层 (Agent Layer)
位于 `agents.py`，负责业务逻辑编排。
- **MainAgent**: 核心路由器。
  - **意图识别**: 基于 LLM 或规则判断用户意图（如“生日关怀”、“节日祝福”、“其他”）。
  - **任务分发**: 将任务调度给具体的 SubAgent 或尝试直接调用 Skill。
- **SubAgents**: 特定领域的业务专家（如 `BirthdayAgent`）。
  - 负责构建特定场景的 Prompt。
  - 调用 LLM 生成文案。
  - 触发后续动作（如生成贺卡）。

### 2.3 技能系统 (Skill System)
位于 `skills/`，是系统的扩展核心。
- **SkillManager (`skills/manager.py`)**: 统一管理所有能力。
  - **自动发现**: 启动时自动扫描 `skills/external/` 目录加载本地 Python 技能。
  - **按需安装**: 支持通过环境变量 (`SKILLS_INDEX_URL`) 或清单 (`SKILLS_MANIFESTS`) 在运行时动态下载远程技能。
  - **自动生成**: 当请求的技能不存在时，利用 LLM 自动生成 Python 代码并热加载执行，实现“自进化”能力。
  - **HTTP 技能**: 支持纯配置型技能（无需代码，直接代理 HTTP 请求）。
- **内置技能**:
  - `profile`: 模拟客户画像查询。
  - `card`: 基于 PIL 生成图片贺卡。
  - `pdf`: 基于 ReportLab 生成 PDF 文件。

### 2.4 基础设施 (Infrastructure)
- **LLMClient (`llm_client.py`)**: 统一的大模型客户端。
  - **多后端支持**: 优先使用 DeepSeek/GLM API，失败或未配置时回退到 Ollama 本地模型。
  - **容错机制**: Ollama 调用失败时支持从 API 模式降级到 CLI 模式。
  - **Mock 模式**: 无任何模型可用时提供静态响应，保证开发环境可用性。

## 3. 关键工作流 (Key Workflows)

### 3.1 意图识别与路由
1. 用户输入：“王总刚提了新车，发个祝贺”
2. **MainAgent** 分析意图 -> 识别为 `Celebration` (喜事庆祝)。
3. 调度 **CelebrationAgent**。
4. Agent 调用 **Profile Skill** 获取“王总”画像。
5. Agent 调用 **LLM** 生成祝贺文案（排除生日词汇）。
6. Agent 调用 **Card Skill** 生成图片。
7. Agent 调用 **PDF Skill** (若指令包含PDF需求) 导出文件。
8. 返回结果给用户。

### 3.2 外部技能按需加载
1. 用户输入：“请把这段话翻译成英文”
2. **MainAgent** 识别意图为 `Other`。
3. 关键词匹配命中 `translate`。
4. **SkillManager** 检查本地是否存在 `translate` 技能。
   - **不存在**: 尝试从 `SKILLS_INDEX_URL` 下载 `translate.json`。
   - **下载成功**: 解析 Manifest，下载代码或配置 HTTP 代理。
   - **下载失败**: 触发 `_autogen_python_skill`，利用 LLM 编写一个临时的翻译技能并加载。
5. 执行 `translate.run(text=...)`。

## 4. 目录结构 (Directory Structure)

```text
iceman/
├── main.py                 # CLI 入口
├── feishu_server.py        # 飞书服务端入口
├── agents.py               # Agent 定义 (MainAgent, SubAgents)
├── llm_client.py           # LLM 客户端 (DeepSeek/Ollama)
├── skills/                 # 技能模块
│   ├── manager.py          # 技能管理器 (加载/注册/自动生成)
│   ├── builtin/            # 内置技能 (card, pdf, profile)
│   ├── external/           # 外部/自动生成技能存放目录
│   └── __init__.py
├── skills.md               # 技能注册表 (元数据)
├── addskills.md            # 技能开发指南
└── tests/                  # 单元测试
```

## 5. 配置管理

系统主要通过环境变量 (`.env`) 进行配置：
- `DEEPSEEK_API_KEY`: 在线模型密钥。
- `GLM_API_KEY`: ZhipuAI GLM 模型密钥。
- `OLLAMA_HOST`: 本地模型地址。
- `SKILLS_INDEX_URL`: 外部技能索引地址（支持逗号分隔多个）。
- `FEISHU_*`: 飞书应用凭证。
